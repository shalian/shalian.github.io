TS -- 类

TS – 类

面向对象编程的三大特点：

  1. 封装（Encapsulation）：将对数据的操作细节隐藏起来，只暴露对外的接口。外界调用端不需要（也不可能）知道细节，就能通过对外提供的接口来访问该对象，
  2. 继承（Inheritance）：子类继承父类，子类除了拥有父类的所有特性外，还有一些更具体的特性。
  3. 多态（Polymorphism）：由继承而产生了相关的不同的类，对同一个方法可以有不同的响应。

  1. TS 的继承和 ES6 的继承写法和特性也相似都是使用 extends
  2. 继承的子类也有 super()

public / private / protected – 公共/私有/受保护的

  1. public 修饰的是在任何地方可见、公有的属性或方法；
  2. private 修饰的是仅在同一类中可见、私有的属性或方法；
  3. protected 修饰的是仅在类自身及子类中可见、受保护的属性或方法

public – 公共

  1. 一些强类型语言是必须要写这些修饰符的，在’TS’默认是 public，也就是说 public 修饰的是可以省略的。
  2. 通过 super 关键字只能访问基类的公共方法和受保护方法，属性是无法访问的。

private – 私有的

当成员被标记成 private 时，它就不能在声明它的类的外部访问，简单的说，只有自己的 class 内部可以访问，自己的 实例 也无法访问，继承的子类也 无法访问 被 private 修饰的内容

  • 私有属性修饰在构造函数
    无法创建实例，但是可以通过静态方法变相创建实例

protected – 受保护的（可以用来禁止创建实例）

  1. 修饰的属性或方法是受保护的，它和 private 类似，区别是它在子类中也是允许被访问的，简言之 子类可以访问 protected 修饰的内容，实例是不可以的。
  2. 修饰的是 constructor 则当前类不能创建实例
  3. 使用 protected 不能直接创建 对象使用属性，和 private 不同他可以在子类中使用，相同点都可以在自身定义类中使用

readonly – 只读属性

可以使用 readonly 关键字将属性设置为只读的。只读属性必须在声明时或构造函数里被初始化。

支持 es6 – getter / setter

抽象类 abstract

  1. 抽象类是不允许被实例化的
  2. 继承抽象类的子类必须去实现抽象类中所有的 抽象方法 和 抽象属性。（子类必须覆盖抽象类的抽象方法）
  3. 含有声明但未实现的方法（也可以包含已实现的方法）
  4. 一个类只能继承一个抽象类
  5. 一旦有了抽象方法，就一定要把这个类声明为抽象类

接口 和 类

接口可以多继承，抽象类不行
抽象类是对类本质的抽象，表达的是 is a 的关系，xxx is a XX；接口是对行为的抽象，表达的是 like a 的关系，xxx like a XX

  1. 接口
    • 不可以被实例化
    • 含有声明但未实现的方法
    • 一个类可以继承多个接口
    • 子类必须实现其声明未实现的方法
    • 所有成员都是默认 Public 的，因此接口中不能有 Private 成员
    • 子类必须实现接口的所有成员
  • 类继承接口（implements）

  1. 抽象类和接口区别
  • 抽象类是类（事物）的抽象，抽象类用来捕捉子类的通用特性，接口是行为的抽象
  • 接口可以被多层实现，而抽象类只能单一继承
  • 接口不具备继承的任何具体特点，仅仅承诺了能够调用的方法
  • 抽象类更多的定义是在一系列紧密相关的类之间，而接口大多数是定义在关系疏松但都实现某一功能的类中
