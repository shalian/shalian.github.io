React 类组件进阶（三）

React 组件进阶

一、Context 跨层级传值

1.1 介绍

Context 提供了一个无需为每层组件手动添加 props，就在组件树间进行数据传递的方法。

在一个典型的 React 应用中，数据是通过 props 属性自上而下(由父及子)进行传递的，但这种做法对于某些类型的属性而言是极其繁琐的(例如: 地区偏好，UI 主题)，这些属性是应用程序中许多组件都需要的。Context 提供了一种在组件之间共享此类值的方式，而不必显式地通过组件树的逐层传递 props.
跨层级传值

1.2 语法

创建 Context

  1. 提供数据

创建的 Context 上提供了一个 Provider 属性，整体作为标签使用，这个标签上提供的一个属性 value，作用是提供数据。

  • <MyContext.Provider value={xxx}></MyContext.Provider>
  1. 接收数据
  • <MyContext.Consumer> {value => } </MyContext.Consumer>（标签形式）
  • Class.contextType（属性形式）

作用：也是获取 Provider 上提供的数据

使用：作为组件的一个属性使用

在组件内部会自动生成一个数据，储存在 this 的 context 属性上。

还有另一种写法 static contextType = MyContext

Class.contextType 和 Context.Consumer 区别

  1. Class.contextType 使用时会自动在组件上添加一个属性 context，储存离当前组件最近的 Provider 提供的数据，不能写多个
  2. Context.Consumer 可以写多个，能获取到所有 Provider 提供的数据
  3. 动态的数据，还可以传递方法

二、错误边界

2.1 介绍

React 中如果组件使用有错误，React 会把页面中的组件全部卸载，会造成页面白屏，这种体验是不好的。

在 React 16 版本后，添加了一个功能，错误边界，错误边界作用：当 React 运行中有错误时，这时显示一个备用的页面，不显示白屏，还可以把报错信息发送到指定的位置。

2.2 定义

错误边界也是一个组件，这个组件必须是 class定义 的，如果组件中定义了 static getDerivedStateFromError() 或 componentDidCatch() 这两个生命周期方法中的任意一个（或两个）时，那么它就变成一个错误边界。当抛出错误后，请使用 static getDerivedStateFromError() 渲染备用 UI ，使用 componentDidCatch() 打印错误信息。

2.3 使用

用错误边界将组件包裹起来就行

四、高阶组件

4.1 介绍

高阶组件(HOC) 是 React 中用于复用组件逻辑的一种高级技巧。HOC 自身不是 React API 的一部分，它是一种基于 React 的组合特性而形成的设计模式。

高阶组件是参数为组件，返回值为新组件的函数。

五、性能优化

5.1 shouldComponentUpdate

作用：是否允许组件更新界面

判断组件自身 state 和 传值 props 是否发生改变，若无变化则不重新渲染

5.2 PureComponent

PureComponent 来定义类形式组件，只有当 props 或 state 发生改变时才会触发组件更新。(react 17 之后添加，不需要自己手动添加判断)

5.3 组件注销

组件注销时要把组件绑定的全局事件，定时器，网络请求，setState 取消掉，否则组件可能会报错，还有可能影响其他组件。

六、路由

路由介绍

单页面特点：只需要加载一次页面，通过局部刷新，可以实现跳转或切换页面

优点：加载速度快，用户体验较好

缺点l：

  • 第一次加载相对于传统较慢
  • 不利于 seo
  • 页面相对复杂

安装

npm i --save react-router-dom
引入：import { Route, BrowserRouter, HashRouter, Switch, Redirect, Link, NavLink } from 'react-router-dom'

  1. BrowserRouter、HashRouter：路由包裹容器
    HashRouter：地址栏加 #
    BrowserRouter：地址栏 / ，不加 #

    HashRouter as Router 取别名

  2. Route：定义路由走向，路由规则
    path：匹配路径
    component：加载指定组件
    exact：精准匹配

  3. Switch：只能匹配一个路由规则

  4. Link：类似 a 标签

  5. NavLink：类似 a 标签，并添加 active 类名
