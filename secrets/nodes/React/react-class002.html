  1. value ：读取数据
  2. onChange ：设置数据

常见的受控组件 ：input、textarea、select

1.2 非受控组件

在大多数情况下，我们推荐使用受控组件来处理表单数据。在一个受控组件中，表单数据是由 React 来管理的。另一种替代方案是使用非受控组件，这时表单数据将有 DOM 节点来处理，可以使用 ref 从 DOM 节点中获取表单数据。

有些组件无法提前知晓它们子组件的具体内容。在 Sidebar (侧边栏)和 Dialog (对话) 等展现通用容器(box)的组件中特别容易遇到这种情况。我们建议这些组件使用一个特殊的 children prop 来将他们的子组件传递到渲染结果中。

四、网络请求

4.1 网络请求方式

方式如下：ajax、jquery、fetch、axios

4.2 fetch 网络请求

介绍

  1. 是 es6 中提供的一种新的网络请求方式。Fetch API 提供了一个 javacript 接口，用于访问和操纵 HTTP 管道的部分，不是 ajax 了
  2. fetch 方法是一个全局方法，可以在任意位置使用
  3. fetch() 返回的 Promise ，使用 .then() 获取请求成功的结果，使用 .catch() 捕获错误信息
  4. 默认情况下，fetch 不会从服务端发送或接收任何 cookies，要发送 cookies，必须设置 credentials 选项，credentials 的默认值是 same-origin

语法

参数说明

  1. url，网络请求地址
  2. 参数 2 是一个很大的配置项
    1. body：要向后台发送的数据，数据格式是 'key=value&key=value'
    2. cache：是否缓存
    3. credentials：是否发生和接收 cookie
    4. headers：请求头，如果是 post 请求，需要配置 {'content-type':'application/x-www-form-urlencoded}
    5. mode：是否允许跨域

返回值

fetch() 返回的 Promise 对象，使用 then() 获取请求成功的结果，这个结果是一个 Response 对象，使用前需要解析，解析方法有:

  • blob() ：解析成二进制
  • json() ：解析成 JSON 对象
  • text() ：解析成文本
