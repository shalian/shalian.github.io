js事件循环

更新于2024-05-05
|字数总计:690|阅读时长:2分钟|阅读量:

JavaScript 是一门单线程语言 (换句话说：一个时间内我只能做一件事)，异步操作都是放到事件循环队列中，等待主执行栈来执行。

JavaScript 是如何执行的 (执行顺序)

从上往下，一行一行执行。如果中间的某一行书写有误。那么程序在运行到这一行会报错并停止向下继续运行。先把所有的同步代码执行完毕，然后再开始执行异步代码。

事件循环（Event Loop）

事件循环(Event Loop)是 JavaScript 引擎处理异步任务的机制。当 JavaScript 引擎遇到异步任务时，会将其放入相应的任务队列中，并继续执行同步代码，直到同步代码执行完毕或遇到下一个异步任务。当前的宏任务执行完毕后，JavaScript 引擎会按照一定的规则从微任务队列中取出任务执行，直到微任务队列为空；然后再从宏任务队列中取出下一个宏任务执行。这个过程就是事件循环。

宏任务与微任务

在 JS 中, 代码分为了两种, 同步和异步。异步任务中也分为了两种, 宏任务与微任务。
_宏任务_：script 整体代码、异步 Ajax 请求、setTimeout、setInterval、requestAnimationFrame、i/o 操作，setImmediate、UI rendering 等…；
_微任务_：Promise.then() catch() finally()、async/await、process.nextTick(NodeJs)、Object.observe、queueMicrotask()、MutationObserver(H5 新特性)…

总结： 宏中有微，先微再宏。遇到 await，后续为微。

执行规则

  • 每遇到 then 就添加至微任务队列。当这个 then 里的代码执行完毕，其下一个 then 才会被添加至微任务队列中。

  • 遇到 await 时，await 后续执行作为微任务添加到微队列

执行顺序

  1. js 先执行完 script 宏任务内所有同步任务，再执行异步任务
  2. 判断异步任务队列是否存在微任务
  3. 若存在微任务，执行完所有微任务后，执行下一个宏任务；反之则直接执行下一个宏任务
  4. 每一个宏任务执行完之后，都会检查是否存在待执行的微任务，如果有，则执行 完所有微任务之后，再继续执行下一个宏任务。
