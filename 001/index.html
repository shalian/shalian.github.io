二、React 事件处理

合并事件：把所有的事件都绑定在 document 上面

2.1 事件绑定方式

  1. 在标签绑定，使用的属性方式
  2. 事件是以 ”on+事件名称“ 组成，使用 小驼峰命名
  3. 事件的功能一般定义在外部，事件这里写 this.事件函数名称
  4. 标签里不能加 () ，表示调用
  1. 事件定义在组件中，和render函数、生命周期函数同级

state -> render() -> 虚拟DOM -> 真实DOM -> 事件 -> setState() -> state

2.2 事件中 this 指向当前组件的方式

事件三要素：

this、事件对象、参数

  1. 普通函数：没有this，undefined

改变this指向：call()、apply()、bind()

  1. 箭头函数：this 指向当前组件实例

三、生命周期

生命周期函数（7个，废掉了3个，还剩4个）

componentWillMount ：组件将要被渲染，还未渲染（17废掉，但还可以使用）

componentDidMount ：组件渲染完成（组件进入运行状态）【网络请求、绑定事件】

shouldComponUpdate ：是否允许组件更新（这个函数中必须有返回值，如果为 true，表示允许更新；false表示不允许更新）

componentWillUpdate ：组件将要开始更新（17废掉，但还可以使用）

componentDidUpdate ：组件更新完成（重新进入允许状态）

componentWillReceiveProps ：组件接收 props 更新

componentWillUnMount ：组件将要卸载，还可以访问 this，this.setDate() 只执行一次【收尾工作，清空定时器、取消事件绑定】

组件有三个生命周期函数只执行一次：

componentWillMount 、componentDidMount 、componentWillUnMount
